# Data Dictionary

This dictionary covers the included raw sample file, the optional public raw file generated by `src/public_data_ingestion.py`, and the processed tabular outputs generated by the pipeline. The sample values are portfolio placeholders that preserve the structure expected from public county-level datasets.

## Files Covered

- `data/raw/maryland_county_health_access_sample.csv`
- `data/raw/maryland_county_health_access_public.csv` when generated locally by `make live-data`
- `data/raw/source_manifest.json`
- `data/raw/public_data_provenance.json` when generated locally by `make live-data`
- `data/processed/dashboard_county_risk.csv`
- `data/processed/model_feature_table.csv`
- `data/processed/model_predictions.csv`
- `data/processed/feature_importance.csv`
- `data/processed/county_ai_summaries.csv`
- `data/processed/state_average.csv`

`model_metrics.json` and `access_risk_model.joblib` are model artifacts rather than row/column datasets.

## Notes for Employers and Reviewers

- Race/ethnicity-style fields are included only for descriptive equity review and should not be used to deny services, target individuals, or infer individual health status.
- The default model target is an external-proxy sample target based on poor/fair health rate. The optional `demo_score` target is derived from the score itself and is useful only for demonstrating modeling workflow.
- In a production refresh, each raw field should be traceable to a specific extract date, table ID, API endpoint, and data license or usage note.
